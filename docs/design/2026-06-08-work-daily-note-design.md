# work--daily-note 设计文档

## 目标

开发一个 Win11 桌面便签应用，用于快速记录每天零散工作事项，并在固定时间生成可复制到禅道的日报文本。

## 范围

包含：

- 快速记录零散事项。
- 今日事项列表查看、编辑、删除。
- 每天 18:00 定时弹出整理窗口。
- 使用 OpenAI 兼容在线 AI 接口整理内容。
- 自动按工作类型分类。
- 生成可复制到禅道的日报文本。
- 一键复制生成结果到剪贴板。
- 本地配置 AI 接口地址、Key、模型名和提醒时间。

不包含：

- 自动登录禅道。
- 自动提交禅道任务或日报。
- 多端同步。
- 团队协作。

## 技术方案

采用 Electron + Node.js 开发 Win11 桌面应用。

原因：Electron 对桌面窗口、托盘、系统通知、剪贴板、本地文件读写支持成熟，适合快速实现这个工具。

## 应用结构

- 主进程：负责窗口、托盘、定时提醒、本地文件读写、剪贴板和 AI 请求代理。
- 渲染进程：负责便签录入、列表管理、整理结果展示和设置界面。
- 本地存储：使用 JSON 文件保存事项和设置。

## 数据设计

事项字段：

- id：唯一标识。
- content：原始记录内容。
- category：工作类型，可为空，由 AI 或用户设置。
- date：所属日期。
- createdAt：创建时间。
- updatedAt：更新时间。

设置字段：

- apiBaseUrl：OpenAI 兼容接口地址。
- apiKey：接口 Key，仅保存在本机配置文件。
- model：模型名。
- reminderTime：提醒时间，默认 18:00。

## 工作分类

默认分类：

- 日常沟通与需求确认
- 问题排查与技术支持
- 开发与代码调整
- 测试验证与发布支持
- 文档、配置与环境维护

## AI 整理逻辑

用户点击“AI 整理”后，应用把当天事项发送给配置的 OpenAI 兼容接口，请求返回：

- 分类后的事项摘要。
- 适合复制到禅道的日报文本。
- 明日计划建议。

AI 请求失败时，界面显示错误，并保留原始事项，不删除任何数据。

## 界面设计

- 记录页：输入事项，回车保存。
- 今日列表：展示当天所有事项，支持编辑和删除。
- 整理页：点击“AI 整理”，显示禅道文本，并支持一键复制。
- 设置页：配置 AI 地址、Key、模型名和提醒时间。

## 定时提醒

应用常驻托盘。到达提醒时间后，弹出整理窗口。用户也可以从托盘菜单手动打开窗口。

## 安全与隐私

- API Key 不写入仓库。
- API Key 只保存在本机用户配置目录。
- 只有点击“AI 整理”时才发送当天事项到在线 AI 接口。
- 不记录明文密钥到日志。

## 验证标准

- Win11 可启动桌面应用。
- 可新增、编辑、删除事项。
- 到 18:00 可弹出整理窗口。
- 可配置 OpenAI 兼容接口并生成日报文本。
- 可一键复制生成结果。
- 关闭窗口后应用仍可在托盘中运行。
