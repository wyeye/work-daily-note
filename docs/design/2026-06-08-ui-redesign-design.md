# UI 与交互重设计方案

日期：2026-06-08
项目：Work Daily Note
范围：Win11 Tauri 桌面便签的 UI、交互、整理能力与本地 JSON 存储结构设计。

## 1. 背景

当前应用用于记录零散工作事项，并生成可复制的日报文本。现有界面是三页签结构：记录、整理、设置。用户希望全面重做视觉与交互，重点提升操作速度、信息清晰度和视觉质感，同时不再在界面中展示“禅道”相关表述。

本方案基于已确认需求、现有项目结构和 UI/UX 检索结果制定。技术栈保持 Tauri + Rust + 原生 HTML/CSS/JS，不引入前端框架。

## 2. 目标

1. 启动后快速记录：默认进入记录页，输入框自动聚焦。
2. 记录方式聊天化：支持一句话保存，也支持 `#项目名` 快速标记。
3. 整理能力增强：一次点击生成日报文本、分类展示、项目展示。
4. 去除界面中的“禅道”表述：统一使用“日报文本 / 复制文本”。
5. 设置页分层：基础配置直接显示，高级设置默认收起。
6. 本地 JSON 存储为未来文件夹同步和增量合并预留结构。

## 3. 非目标

1. 本次不实现账号体系或云端同步服务。
2. 本次不自动登录或提交任何第三方系统。
3. 本次不引入 React、Vue 等前端框架。
4. 旧数据不重点迁移；新版本需保证数据正确，旧字段缺失时界面不崩溃。

## 4. 已确认需求

- 使用节奏：随手记 + 下班集中整理。
- 优先级：操作速度 > 信息清晰 > 视觉质感。
- 默认入口：应用启动或托盘打开后进入记录页。
- 记录方式：聊天式输入。
- 标签/项目：支持 `#项目名`，也支持保存后编辑标签；不输入项目也可保存。
- 项目识别：用户写的 `#项目名` 优先，其余由 AI 自动识别。
- 整理触发：点击一个“开始整理”按钮，同时生成三类结果。
- 整理结果：三个页签切换，默认显示“日报文本”。
- 日报文本默认格式：今日完成 / 明日计划。
- 提醒行为：提醒时间到后先显示今日事项确认，再点击开始整理。
- 设置页：基础设置直接显示；日报模板、项目识别规则、启动页、Enter 行为、提醒策略放入高级设置。
- 视觉方向：清爽效率风。
- 存储方向：多 JSON 文件 + 变更日志，便于未来通过文件同步工具同步。

## 5. 信息架构

保留三个主页签：

1. 记录
2. 整理
3. 设置

### 5.1 记录页

记录页是默认页。页面打开后，光标自动进入底部输入框。

核心区域：

- 聊天式事项流
- 底部一句话输入框
- 常用标签提示
- 今日统计与整理入口

用户可直接输入一句话保存，也可输入 `#项目名 内容`。保存后事项进入聊天流，按时间显示。

### 5.2 整理页

整理页包含一个主要按钮：“开始整理”。点击后同时生成：

1. 日报文本
2. 分类展示
3. 项目展示

结果以页签切换，默认打开“日报文本”。

### 5.3 设置页

基础设置直接展示：

- AI 接口地址
- API Key
- 模型名
- 提醒时间

高级设置默认收起：

- 日报模板
- 项目识别规则
- 启动页
- Enter 保存/换行行为
- 提醒策略

## 6. 记录页交互

### 6.1 输入

- 输入一句话即可保存。
- 支持 `#项目名` 语法。
- 不写 `#项目名` 也能保存。
- Enter 保存，Shift+Enter 换行。
- 按钮保存也可用。
- 保存成功后清空输入框，并保持焦点。
- 保存失败时保留输入内容。

### 6.2 事项展示

事项以聊天流展示，字段包括：

- 时间
- 内容
- 项目/标签
- 操作按钮

每条事项支持：

- 编辑内容
- 编辑标签/项目
- 删除

删除前需要确认。删除后 UI 隐藏该事项，但存储层保留删除标记，便于未来同步合并。

### 6.3 空状态

无事项时显示简洁引导：输入第一条事项。空状态不展示复杂说明。

## 7. 整理页交互

### 7.1 生成流程

点击“开始整理”后：

1. 检查今日事项是否为空。
2. 检查 AI 基础配置是否可用。
3. 向 AI 请求结构化整理结果。
4. 同时填充日报文本、分类展示、项目展示。
5. 默认展示日报文本页签。

### 7.2 结果内容

#### 日报文本

默认格式：

```text
今日完成
1. ...

明日计划
1. ...
```

提供“复制文本”按钮。

#### 分类展示

按工作类型展示摘要和来源事项数量。分类用于理解当天工作分布。

#### 项目展示

按项目展示摘要和来源事项。项目来源优先级：

1. 用户输入的 `#项目名`
2. AI 自动识别
3. 未分项目

### 7.3 事项变化提示

整理结果保存时记录 `sourceRevisionHash`。如果事项在生成后发生变化，整理页提示“事项已变化，可重新整理”。

## 8. 设置与提醒

### 8.1 设置页

基础配置直接可见，保存后立即生效。API Key 不完整展示。

高级设置默认收起，避免干扰日常使用。高级项包括：

- 日报模板
- 项目识别规则
- 启动页
- Enter 行为
- 提醒策略

### 8.2 提醒确认窗口

到提醒时间后显示今日事项确认窗口。窗口包含：

- 今日事项列表
- “开始整理”按钮
- “稍后”按钮

点击“开始整理”后进入整理页并触发整理流程。点击“稍后”关闭确认窗口，不自动整理。

## 9. 视觉系统

使用清爽效率风，适合桌面效率工具。

### 9.1 颜色

| 用途 | 颜色 |
| --- | --- |
| 主色 | `#0D9488` |
| 辅助色 | `#14B8A6` |
| 行动色 | `#F97316` |
| 背景 | `#F0FDFA` |
| 文本 | `#134E4A` |
| 卡片 | `#FFFFFF` |

行动色只用于保存、开始整理、复制等关键动作。

### 9.2 字体

使用系统字体优先，保证 Windows WebView2 中显示清晰。推荐字体栈：

```css
font-family: "Segoe UI", "Microsoft YaHei", sans-serif;
```

### 9.3 组件

核心组件：

- 顶部页签
- 聊天事项卡片
- 底部输入条
- 标签/项目标记
- 整理结果页签
- 状态提示
- 设置表单
- 提醒确认窗口

### 9.4 动效

- 使用 150–300ms 颜色、边框、阴影变化。
- 不使用会造成布局变化的缩放动画。
- 尊重 `prefers-reduced-motion`。

### 9.5 可访问性

- 所有按钮有可见焦点状态。
- 错误提示使用 `aria-live`。
- 不只依赖颜色表达状态。
- 文本对比度满足 WCAG AA。
- 图标不用 emoji，统一使用 SVG。

## 10. 技术结构

技术栈保持：

- Tauri 2
- Rust
- 原生 HTML/CSS/JS

### 10.1 渲染层模块

建议将现有渲染逻辑拆成以下职责：

- 路由与状态管理
- 记录页渲染
- 整理页渲染
- 设置页渲染
- Toast/错误提示
- 标签解析工具

### 10.2 Rust 层职责

Rust 继续负责：

- 本地 JSON 存储
- 提醒调度
- AI 请求
- 剪贴板写入
- 窗口/托盘行为

## 11. 本地 JSON 存储设计

未来可能通过 OneDrive、坚果云、Syncthing 等文件同步工具同步应用数据。因此不再依赖单一大 JSON 文件，改为多 JSON 文件 + 变更日志结构。

### 11.1 目录结构

```text
sync-data/
  notes/
    2026-06-08.json
    2026-06-09.json
  results/
    daily/
      2026-06-08.json
  settings/
    appearance.json
    interaction.json
    organizer.json
    ai.json
    reminder.json
  changes/
    <deviceId>/
      2026-06.jsonl

local-data/
  device.json
  secrets.json
  window.json
  cache/
```

### 11.2 可同步数据

可放入 `sync-data/`：

- 事项
- 日报结果
- 主题/视觉偏好
- 启动页
- Enter 行为
- 日报模板
- 项目识别规则
- AI 接口地址
- 模型名
- 提醒时间

### 11.3 本机数据

只放入 `local-data/`：

- API Key
- `deviceId`
- 窗口大小/位置
- 托盘状态
- 提醒是否在本机启用
- 临时缓存

API Key 默认不进入同步目录。

### 11.4 事项文件

每日事项文件示例：

```json
{
  "schemaVersion": 1,
  "date": "2026-06-08",
  "notes": [
    {
      "id": "note_01",
      "content": "#移动端 调整列表筛选",
      "tags": ["移动端"],
      "project": "移动端",
      "createdAt": "2026-06-08T11:10:00+08:00",
      "updatedAt": "2026-06-08T11:10:00+08:00",
      "deletedAt": null,
      "revision": 1,
      "updatedBy": "device_01"
    }
  ]
}
```

### 11.5 生成结果文件

单日结果文件示例：

```json
{
  "schemaVersion": 1,
  "date": "2026-06-08",
  "dailyText": "",
  "categorySummaries": [],
  "projectSummaries": [],
  "sourceNoteIds": [],
  "sourceRevisionHash": "",
  "templateVersion": 1,
  "promptVersion": 1,
  "model": "gpt-4.1-mini",
  "generatedAt": "",
  "editedAt": "",
  "updatedAt": "",
  "updatedBy": "",
  "revision": 1
}
```

规则：

- 日报文本、分类展示、项目展示都保存到每日结果文件。
- 用户手动改过的日报文本视为重要数据，需要同步。
- 不默认保存完整 AI 原始响应，避免文件膨胀和敏感内容增加。
- 保存 `sourceRevisionHash`，用于判断事项变化后是否需要重新整理。
- 同一天多设备都改了日报时，不互相覆盖，生成冲突副本，例如：`2026-06-08.conflict.<deviceId>.json`。

### 11.6 变更日志

`changes/<deviceId>/*.jsonl` 记录追加式变更。记录类型包括：

- `note_created`
- `note_updated`
- `note_deleted`
- `result_generated`
- `result_edited`
- `result_deleted`
- `setting_updated`

示例：

```json
{"entityType":"note","entityId":"note_01","operation":"note_updated","revision":2,"changedAt":"2026-06-08T12:00:00+08:00"}
```

### 11.7 写入规则

- 写 JSON 时先写临时文件，再替换正式文件，避免文件同步工具读到不完整内容。
- 删除数据先写 `deletedAt`，不立即移除。
- UI 默认隐藏已删除数据。
- 未来完成同步合并后，可清理过期删除记录。

## 12. AI 数据流

整理请求发送给 AI 的内容包含：

- 当天未删除事项
- 用户输入的项目/标签线索
- 当前日报模板
- 分类与项目整理要求

AI 返回结构化 JSON，字段包括：

- `dailyText`
- `categorySummaries`
- `projectSummaries`
- `tomorrowPlan`

渲染层只展示整理后的结构化内容。AI 请求失败或返回格式异常时，界面展示错误，不清空事项。

## 13. 错误处理与状态反馈

### 13.1 记录页

- 空内容不保存。
- 保存失败时保留输入内容。
- 删除前显示确认。

### 13.2 整理页

- 无事项时不请求 AI。
- AI 配置缺失时提示去设置页补充。
- 请求中显示“整理中”，按钮禁用。
- AI 返回异常时显示错误，不清空已有事项。
- 复制成功/失败都给出明确提示。

### 13.3 设置页

- 提醒时间格式不合法时回到默认值。
- API Key 不完整展示。
- 保存失败时保留用户输入。

### 13.4 提醒窗口

- “稍后”关闭确认窗口，不自动整理。
- “开始整理”进入整理页并触发整理流程。

## 14. 验证范围

提交前执行：

```bash
npm run check
git diff --check
git status --short
```

使用 `/tmp` 下的 `python3` 临时脚本验证：

- `#项目名` 解析
- JSON 结构读写
- 缺失字段兼容

Win11 手动验证：

- 应用可启动。
- 启动默认进入记录页。
- 可一句话记录、编辑、删除事项。
- 到提醒时间显示确认窗口。
- 整理页可生成三类结果。
- 复制文本可写入剪贴板。
- 设置可保存。
- 关闭窗口后托盘仍可用。

## 15. UI/UX 依据

使用 `ui-ux-pro-max` 检索得到的建议：

- 产品类型：桌面效率工具。
- 推荐方向：Minimal Single Column、AI-Native UI、清爽效率风。
- 推荐配色：青绿色主色 + 橙色行动色。
- 推荐字体：Inter/系统无衬线；本项目采用 Windows 系统字体优先。
- 注意事项：高对比文本、可见焦点、错误 `aria-live`、动画尊重 `prefers-reduced-motion`、按钮有明确悬停状态。
